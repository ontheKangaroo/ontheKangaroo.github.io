# Chapter 1: Introduction

# 1. Language Processors

Different Ways

  • source program -> Compiler -> target program.

    • faster

  • source program + input -> Interpreter -> output.

    • errpr diagnostics: statement by statement

  • source program -> Translator -> intermediate program; intermediate program + input -> Virtual machine -> output. (JAVA)

  • source program -> Preprocessor -> modified source program -> Compiler -> target assembly program -> Assembler -> relocatable machine code; relocatable machine code + library files + relocatable object files -> Linker / Loader -> target machine code.

    • separate files

    • compiled in pieces

# 3. The Evolution of Programming Languages

# 3.1 The Mode to Higher-level Languages

  • Generation

    • First: machine languages

    • Second: assembly languages

    • Third: higher-level languages - Fortran, Cobol, Lisp, C, C++, C#

    • Fourth: for specific applications - NOMAD, SQL, Postscript

    • Fifth: logic- and constraint-based languages - Prolog, OPS5

  • Imperative / Declarative

    • Imperative: C, C++, C#, Java

      • there is a notion of program state and statements that change the state

    • Declarative: ML, Haskell, Prolog

  • von Neumann Language: computational model is based on the von Neumann computer architecture - Fortran, C

  • Object-oriented language: a program consists of a colletion of objects that interact with one another - Simula 67, Smalltalk, C++, C#, Java, Ruby

  • Scripting languages: "gluing together" - Awk, JS, Perl, PHP, Python, Ruby, Tcl

# 6. Programming Langauge Basics

# 6.1 The Static / Dynamic Distinction

  • static: allows the compiler to decide an issue / the issue can be decided at compile time

    • static scope / lexical scope

  • dynamic: only allows a decision to be made when excuting the program / require a decision at run time

    • dynamic scope: e.g. public static int x;

# 6.2 Environmnets and States

e.g. if x is not static, its value can be change at run time

  • environment: names -> locations

  • state: locations -> values

# 6.3 Static Scope and Block Structure

  • static scope: C, C++, Java, C# (C family) for public , private , and protected

    • C's static-scope policy

      • A C program consists of a sequence of top-level declararions of variables and functions

      • Functions may have variable declarations within them, where variables include local variables and parameters. The scope of each such delaration is restricted to the function in which it happens

      • The scope of a top-level declaration of a name x consists of the entire program that follows, with the exception of those statements that lie within a function that also has a declaration of x

  • blocks
